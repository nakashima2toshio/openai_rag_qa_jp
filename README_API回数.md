 API呼び出し回数の概算

1. cc_news (英語ニュース)

- 文書数: 約497件（CSVから497件の有効データ）
- チャンク数: 約500-600個（1文書あたり1-2チャンク）
- API呼び出し回数: 約500-600回
- バッチサイズ5で処理するため、実際のAPI呼び出しは同じ

2. japanese_text (日本語Webテキスト)

- 文書数: 約961件
- チャンク数: 約966個（既に確認済み）
- API呼び出し回数: 約966回

3. wikipedia_ja (Wikipedia日本語版)

- 文書数: 約21,400件（214,132行 ÷ 10行/記事）
- チャンク数: 約42,800個（1文書あたり平均2チャンク）
- API呼び出し回数: 約42,800回
- ⚠️ 注意: 大量のAPI呼び出しになるため、コストに注意

コスト見積もり（gpt-5-miniの場合）：

- 入力: $0.15/1Mトークン
- 出力: $0.60/1Mトークン
- 1回あたり平均: 入力500トークン + 出力200トークン

概算コスト:
- cc_news: 約$0.25
- japanese_text: 約$0.50
- wikipedia_ja: 約$22（要注意！）
